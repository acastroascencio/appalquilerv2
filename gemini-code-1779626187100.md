# 🗄️ Configuración de Base de Datos (Supabase SQL)

Este esquema garantiza que cada administrador solo vea sus propios datos mediante RLS (Row Level Security).

## Tablas Principales

### 1. `perfiles`
- `id`: uuid (references auth.users) - PK
- `nombre_completo`: text
- `datos_pago`: jsonb (Cuentas BCP, Interbank, Yape/Plin)
- `creado_el`: timestamp

### 2. `departamentos`
- `id`: uuid - PK
- `admin_id`: uuid (references auth.users)
- `identificador`: text (Ej: "Dpto 101")
- `precio_alquiler`: numeric
- `configuracion`: jsonb (habitaciones, baño_propio, tiene_cocina)
- `fotos`: text[] (URLs de Supabase Storage)

### 3. `inquilinos`
- `id`: uuid - PK
- `admin_id`: uuid (references auth.users)
- `depto_id`: uuid (references departamentos)
- `nombre`: text
- `celular`: text
- `moto_info`: jsonb (placa, monto_asociacion)
- `documentos`: jsonb (dni_url, contrato_url)

### 4. `lecturas_mensuales`
- `id`: uuid - PK
- `admin_id`: uuid (references auth.users)
- `inquilino_id`: uuid (references inquilinos)
- `mes`: text (Ej: "Mayo 2026")
- `luz`: jsonb (anterior, actual, costo_unitario)
- `agua`: jsonb (anterior, actual, costo_unitario)
- `seguridad`: numeric
- `pagado`: boolean (default: false)
